How to Host Your Own Website for Free (Using GitHub)

Stop paying for hosting. Here is how I built this blog in 5 minutes using GitHub Pages.

If you are tired of restrictive platforms like Blogger or Wix, there is a "secret" way to host a website that gives you total control, zero ads, and costs $0. It's called GitHub Pages.

You don't need to be a coding genius to use it. If you can drag and drop a file, you can publish a site. Here is the exact process I used to launch this blog.

1 Create a GitHub Account

Go to github.com and sign up. It is the platform developers use to store code, but we are going to use it to store our blog posts.

2 Create a Repository

Think of a "Repository" (or Repo) as a folder for your website project.

  • Click the + icon in the top right corner.
  • Select New repository.
  • Name it something simple (e.g., my-ai-blog).
  • Check the box that says Add a README file (this makes setup easier).
  • Click Create repository.

3 Upload Your Home Page

Every website needs a "Main Entrance." In the web world, this file must be named index.html.

  • In your new repository, click Add file > Upload files.
  • Drag your HTML file (the one you wrote) into the box.
  • Crucial: Ensure this file is named exactly index.html.
  • Scroll down and click the green button: Commit changes.

4 Go Live!

Now we tell GitHub to turn that code into a website.

  • Click the Settings tab at the top of your repository page.
  • On the left sidebar, click Pages.
  • Under "Branch", click the dropdown that says "None" and select main.
  • Click Save.

Wait about 60 seconds. Refresh the page, and you will see a bar at the top saying: "Your site is live at..."

How to Add More Pages (Navigation)

Once your homepage is live, you'll want to add more articles. Here is the trick:

1. Create a new HTML file for your new article (e.g., viral_photos_post.html).
2. Upload it to the same repository.
3. Add the Navigation Menu (shown at the top of this page) to the top of every file so people can click between them.
